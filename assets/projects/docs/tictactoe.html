Tic Tac Toe AI with Minimax

Project #3 Published: 18/08/2025

This project implements the classic game of Tic Tac Toe with an unbeatable AI opponent. The AI is built on the Minimax algorithm, improved with Alpha-Beta pruning for efficiency. Despite the simplicity of Tic Tac Toe, the project is an excellent introduction to game tree search, a fundamental technique in AI.

  • Unbeatable AI using perfect play
  • Support for both Player vs Player and Player vs AI
  • Configurable AI side (X or O)
  • Optional move suggestions (highlighting the best move)
  • Clean board rendering with coordinate labels

üß† The Minimax Algorithm

Minimax is a recursive search algorithm used to decide the best move in two-player games. The idea is simple:

  • Assume both players play optimally.
  • The maximizing player (the AI) tries to maximize the score.
  • The minimizing player (the opponent) tries to minimize the score.

At each turn, the algorithm explores all possible moves, simulates the outcome, and assigns a score: win (+10), loss (‚àí10), or draw (0). These scores propagate upwards in the game tree, allowing the AI to pick the move that guarantees the best final outcome. 

SearchResult MiniMax(int board[9], int turn, int depth, int alpha, int beta) {
int win = is_win(board);
if (win) return (SearchResult){ (win == AI ? 10 - depth : depth - 10), -1 };
if (is_board_filled(board)) return (SearchResult){0, -1};

SearchResult best = { (turn == AI ? -11 : 11), -1 };
for (int move = 0; move < 9; move++) {
    if (board[move] != EMPTY) continue;
    board[move] = turn;
    SearchResult result = MiniMax(board, (turn==1?2:1), depth+1, alpha, beta);
    board[move] = EMPTY;

    if ((turn == AI && result.value > best.value) ||
        (turn != AI && result.value < best.value)) {
        best.value = result.value;
        best.action = move;
    }

    if (turn == AI) alpha = (best.value > alpha) ? best.value : alpha;
    else beta = (best.value < beta) ? best.value : beta;
    if (beta <= alpha) break; // pruning
}
return best;
}

‚úÇÔ∏è Alpha-Beta Pruning

While Minimax explores every possible game state, many branches are unnecessary because optimal play would never reach them. Alpha-Beta pruning introduces two bounds:

  • Alpha: the best score the maximizer is guaranteed so far.
  • Beta: the best score the minimizer is guaranteed so far.

If at any point beta ‚â§ alpha, the current branch can be cut off safely ‚Äî no need to explore further, as a better move is already known. This reduces the search space dramatically without changing the result.
